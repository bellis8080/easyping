# Story 1.3: User Authentication (Email/Password & OAuth)

## Status
Approved

## Story

**As an** end user,
**I want** to sign up and log in with email/password or Google OAuth,
**so that** I can access EasyPing securely.

## Acceptance Criteria

1. Sign-up page with email/password form (with Zod validation)
2. Login page with email/password form
3. "Forgot password" flow with email reset link
4. Google OAuth sign-in button (using Supabase Auth)
5. After successful auth, user redirected to dashboard
6. Supabase Auth session management configured (refresh tokens, persistent sessions)
7. Protected routes middleware that redirects unauthenticated users to login
8. User profile stored in database with tenant_id set to default organization
9. **Email service configured for password reset and notifications:**
   - **Option 1 (Default):** Use Supabase Auth built-in email for password resets (zero config, SMTP via Supabase)
   - **Option 2 (Production):** External SMTP provider configuration (SendGrid, Mailgun, AWS SES) for custom branded emails
   - **Environment variables documented:** `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_FROM_EMAIL`
   - **Fallback behavior:** If external SMTP fails, fall back to in-app notifications only
   - **Email templates created:** Password reset, welcome email, email verification

### Authentication Architecture

**[Source: docs/architecture/high-level-architecture.md]**

**Authentication Flow:**

1. **Sign-Up Flow:**
   - User submits email/password via sign-up form
   - Next.js calls Supabase Auth `signUp()` API
   - Supabase creates user in `auth.users` table
   - Application creates corresponding record in `users` table with `tenant_id`
   - User redirected to dashboard, session cookie set

2. **Login Flow:**
   - User submits email/password via login form
   - Next.js calls Supabase Auth `signInWithPassword()` API
   - Supabase validates credentials, returns JWT session
   - Session stored in HTTP-only cookie via `@supabase/ssr`
   - User redirected to dashboard

3. **OAuth Flow:**
   - User clicks "Sign in with Google" button
   - Next.js calls Supabase Auth `signInWithOAuth({ provider: 'google' })`
   - Supabase redirects to Google OAuth consent screen
   - Google redirects back to `/auth/callback` with authorization code
   - Supabase exchanges code for access token, creates session
   - Application creates user record if first-time login
   - User redirected to dashboard

4. **Protected Routes Middleware:**
   - Middleware intercepts requests to protected routes
   - Retrieves session from cookies using `@supabase/ssr`
   - Validates JWT token with Supabase
   - Fetches user's `tenant_id` from `users` table
   - Calls `set_tenant_context(tenant_uuid)` to enable RLS
   - Allows request to proceed if authenticated, else redirects to `/login`

**Session Management:**
- Sessions stored as HTTP-only cookies (not localStorage for security)
- Session expiry: 7 days (configurable in Supabase)
- Automatic refresh token rotation
- Middleware refreshes session on each request

**Multi-Tenant Context:**
- **CRITICAL:** After authentication, middleware MUST call `set_tenant_context()` before any database queries
- RLS policies enforce tenant isolation using `current_setting('app.tenant_id')`
- All database queries automatically filtered by tenant

**Users Table Schema:**
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  tenant_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  email TEXT NOT NULL,
  full_name TEXT NOT NULL,
  avatar_url TEXT,
  role TEXT NOT NULL DEFAULT 'end_user',

  last_seen_at TIMESTAMPTZ,
```

```sql
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
```

**Middleware Client (apps/web/src/lib/supabase/middleware.ts):**
```typescript
import { createServerClient } from '@supabase/ssr';
import { NextResponse, type NextRequest } from 'next/server';

export async function updateSession(request: NextRequest) {
  let response = NextResponse.next({
    request: {
      headers: request.headers,
    },
  });

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        get(name: string) {
          return request.cookies.get(name)?.value;
        },
        set(name: string, value: string, options: any) {
          response.cookies.set({ name, value, ...options });
        },
        remove(name: string, options: any) {
          response.cookies.set({ name, value: '', ...options });
        },
      },
    }
  );

  // Refresh session automatically
  const { data: { user } } = await supabase.auth.getUser();

  // If authenticated, set tenant context for RLS
  if (user) {
    const { data: userProfile } = await supabase
      .from('users')
      .select('tenant_id')
      .eq('id', user.id)
      .single();

    if (userProfile) {
      await supabase.rpc('set_tenant_context', {
        tenant_uuid: userProfile.tenant_id
      });
    }
  }

  return response;
}
```

**Main Middleware (apps/web/src/middleware.ts):**
```typescript
import { NextResponse, type NextRequest } from 'next/server';
import { updateSession } from '@/lib/supabase/middleware';

export async function middleware(request: NextRequest) {
  // Refresh session and set tenant context
  const response = await updateSession(request);

  // Check if user is authenticated
  const user = response.headers.get('x-user-id'); // Set by updateSession

  // Protected routes
  const protectedPaths = ['/dashboard', '/tickets', '/kb', '/settings'];
  const isProtectedPath = protectedPaths.some(path =>
    request.nextUrl.pathname.startsWith(path)
  );

  // Redirect to login if accessing protected route without auth
  if (isProtectedPath && !user) {
    const loginUrl = new URL('/login', request.url);
    loginUrl.searchParams.set('redirect', request.nextUrl.pathname);
    return NextResponse.redirect(loginUrl);
  }

  return response;
}

export const config = {
  matcher: [
    '/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)',
  ],
};
```

### Critical Implementation Notes

**IMPORTANT:**

1. **Tenant Context MUST be Set:**
   - After authentication, call `set_tenant_context(tenant_uuid)` in middleware
   - Without this, RLS policies will block all database queries
   - User's tenant_id must be fetched from `users` table after auth

2. **First User Owner Role:**
   - First user to sign up in an organization automatically assigned 'owner' role
   - Check if organization has any users before setting role
   - Subsequent users default to 'end_user' role

3. **OAuth Profile Creation:**
   - Google OAuth does not require password
   - Extract `email` and `full_name` from Google profile
   - Avatar URL can be fetched from Google profile picture
   - Create user record in `users` table on first OAuth login

4. **Session Cookie Security:**
   - Use HTTP-only cookies (not localStorage) to prevent XSS attacks
   - Set `SameSite=Lax` for CSRF protection
   - Cookies should be secure (HTTPS only) in production

5. **Email Configuration:**
   - Default to Supabase built-in email for MVP (zero config)
   - External SMTP configuration optional for production
   - Document both options in README

6. **Password Reset Security:**
   - Reset tokens expire after 1 hour (Supabase default)
   - Tokens are single-use only
   - Always validate token before allowing password update
