# Story 1.6: First-Run Setup Wizard

## Status

Done

## Story

**As a** first-time user,
**I want** a setup wizard on initial deployment,
**so that** I can configure my organization and admin account.

## Acceptance Criteria

1. On first run, detect if organization exists; if not, show setup wizard
2. Setup wizard collects: Organization name, admin email, admin password, AI provider API key (optional)
3. Wizard creates organization record and first user with owner role
4. AI provider configuration saved to database (encrypted API key)
5. After setup, user redirected to main dashboard
6. Setup wizard only appears once (flag in database tracks completion)
7. Wizard UI matches EasyPing branding (clean, modern, ping.me theme)

### Encryption Implementation

**[Source: docs/architecture/database-schema.md#extensions-and-setup]**

PostgreSQL pgcrypto extension is already enabled:
```sql
CREATE EXTENSION IF NOT EXISTS "pgcrypto";
```

**Encryption SQL Functions (New Migration):**
```sql
-- Encrypt data using AES-256
CREATE OR REPLACE FUNCTION encrypt_data(data TEXT, key TEXT)
RETURNS TEXT AS $$
BEGIN
  RETURN encode(pgp_sym_encrypt(data, key), 'base64');
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- Decrypt data
CREATE OR REPLACE FUNCTION decrypt_data(encrypted_data TEXT, key TEXT)
RETURNS TEXT AS $$
BEGIN
  RETURN pgp_sym_decrypt(decode(encrypted_data, 'base64'), key);
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

**TypeScript Encryption Client:**
```typescript
// lib/crypto.ts
import { createClient } from '@supabase/supabase-js';

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
);

export async function encrypt(plaintext: string): Promise<string> {
  const key = process.env.ENCRYPTION_KEY;
  if (!key) {
    throw new Error('ENCRYPTION_KEY environment variable not set');
  }

  const { data, error } = await supabase.rpc('encrypt_data', {
    data: plaintext,
    key: key,
  });

  if (error) throw error;
  return data;
}

export async function decrypt(encryptedData: string): Promise<string> {
  const key = process.env.ENCRYPTION_KEY;
  if (!key) {
    throw new Error('ENCRYPTION_KEY environment variable not set');
  }

  const { data, error } = await supabase.rpc('decrypt_data', {
    encrypted_data: encryptedData,
    key: key,
  });

  if (error) throw error;
  return data;
}
```

**Environment Variable:**
- `ENCRYPTION_KEY` - 32-byte random string for AES-256 encryption
- Generate with: `openssl rand -base64 32`
- Must be same across all deployments to decrypt existing keys
- Store securely (never commit to git)

### Middleware Setup Detection

**[Source: docs/architecture/source-tree.md#nextjs-middleware]**

Next.js middleware runs on every request and can redirect unauthenticated users.

**Setup Check Logic:**
```typescript
// middleware.ts
import { createServerClient } from '@supabase/ssr';
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export async function middleware(request: NextRequest) {
  const response = NextResponse.next();

  // Create Supabase client
  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        get: (name) => request.cookies.get(name)?.value,
        set: (name, value, options) => {
          response.cookies.set({ name, value, ...options });
        },
        remove: (name, options) => {
          response.cookies.delete({ name, ...options });
        },
      },
    }
  );

  // Check if setup is complete (organization exists)
  const { count } = await supabase
    .from('organizations')
    .select('*', { count: 'exact', head: true });

  const isSetupComplete = count > 0;
  const isSetupRoute = request.nextUrl.pathname.startsWith('/setup');

  // Redirect to setup if not complete and not already on setup page
  if (!isSetupComplete && !isSetupRoute) {
    return NextResponse.redirect(new URL('/setup', request.url));
  }

  // Redirect away from setup if already complete
  if (isSetupComplete && isSetupRoute) {
    return NextResponse.redirect(new URL('/dashboard', request.url));
  }

  return response;
}

export const config = {
  matcher: ['/((?!api|_next/static|_next/image|favicon.ico).*)'],
};
```

**Important:**
- Middleware runs on EVERY request, so cache setup status if possible
- Use service role key in middleware for admin checks (bypasses RLS)
- Exclude `/api`, `/_next`, and static assets from matcher

### API Route Implementation

**[Source: docs/architecture/api-specification.md#nextjs-api-routes]**

**Setup API Route Structure:**
```typescript
// app/api/setup/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { createClient } from '@supabase/supabase-js';
import { encrypt } from '@/lib/crypto';
import { z } from 'zod';

const setupSchema = z.object({
  organization: z.object({
    name: z.string().min(1),
    domain: z.string().optional().nullable(),
  }),
  admin: z.object({
    email: z.string().email(),
    password: z.string().min(8),
    fullName: z.string().min(1),
  }),
  aiConfig: z.object({
    provider: z.enum(['openai', 'anthropic', 'azure', 'skip']),
    apiKey: z.string().optional(),
    model: z.string().optional(),
  }),
});

export async function POST(request: NextRequest) {
  try {
    const body = await request.json();
    const data = setupSchema.parse(body);

    const supabase = createClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.SUPABASE_SERVICE_ROLE_KEY! // Admin privileges
    );

    // Check setup not already complete
    const { count } = await supabase
      .from('organizations')
      .select('*', { count: 'exact', head: true });

    if (count > 0) {
      return NextResponse.json(
        { error: 'Setup already completed' },
        { status: 400 }
      );
    }

    // Encrypt AI API key if provided
    let encryptedKey = null;
    if (data.aiConfig.provider !== 'skip' && data.aiConfig.apiKey) {
      encryptedKey = await encrypt(data.aiConfig.apiKey);
    }

    // Create organization
    const { data: org, error: orgError } = await supabase
      .from('organizations')
      .insert({
        name: data.organization.name,
        domain: data.organization.domain,
        settings: data.aiConfig.provider !== 'skip' ? {
          ai_provider: {
            provider: data.aiConfig.provider,
            api_key_encrypted: encryptedKey,
            model: data.aiConfig.model || getDefaultModel(data.aiConfig.provider),
          },
        } : {},
      })
      .select()
      .single();

    if (orgError) throw orgError;

    // Create admin user
    const { data: authUser, error: authError } = await supabase.auth.signUp({
      email: data.admin.email,
      password: data.admin.password,
      options: {
        data: {
          full_name: data.admin.fullName,
          tenant_id: org.id,
        },
      },
    });

    if (authError) throw authError;

    // Create user profile
    const { error: profileError } = await supabase
      .from('users')
      .insert({
        id: authUser.user.id,
        tenant_id: org.id,
        email: data.admin.email,
        full_name: data.admin.fullName,
        role: 'owner',
      });

    if (profileError) throw profileError;

    return NextResponse.json({ success: true, organizationId: org.id });
  } catch (error) {
    console.error('Setup error:', error);
    return NextResponse.json(
      { error: 'Setup failed' },
      { status: 500 }
    );
  }
}

function getDefaultModel(provider: string): string {
  switch (provider) {
    case 'openai': return 'gpt-3.5-turbo';
    case 'anthropic': return 'claude-3-haiku-20240307';
    case 'azure': return 'gpt-35-turbo';
    default: return '';
  }
}
```

**Error Handling:**
- Return 400 if setup already complete
- Return 400 if validation fails (Zod)
- Return 500 if database error
- Log all errors for debugging
- Return user-friendly error messages

**✅ RESOLVED - pgcrypto Extension in Docker (FIXED with Option A):**

**Original Problem:** The Supabase postgres Docker image (`supabase/postgres:15.1.0.147`) had a custom hook system that prevented `CREATE EXTENSION pgcrypto` from executing during init scripts. The hook tried to call `pg_read_file()` on `/etc/postgresql-custom/extension-custom-scripts/before-create.sql` but encountered a permission denied error.

**Root Cause:** The Supabase image's extension creation hook executed in a security context that didn't inherit the postgres role's file read permissions. This was a known limitation when using the Supabase postgres image outside of the Supabase CLI environment.

**Solution Implemented:** Switched from `supabase/postgres:15.1.0.147` to `ankane/pgvector:v0.5.1` (Option A).

**Error Message:**

```sql
ERROR:  permission denied for function pg_read_file
CONTEXT:  PL/pgSQL function inline_code_block line 3 at EXECUTE
SQL statement "do $$
begin
  execute
    replace(
      replace(
        replace(
          replace(
            pg_read_file('/etc/postgresql-custom/extension-custom-scripts/before-create.sql'),
            '@extname@', '''pgcrypto'''
          ),
          '@extschema@', '''public'''
        ),
        '@extversion@', 'null'
      ),
      '@extcascade@', 'false'
    );
exception
  when undefined_file then
    -- skip
end
$$"
```

**Attempted Solutions (all failed):**

1. ✗ Granting `pg_read_server_files` role to postgres (already had it)
2. ✗ Creating extension in different schemas (public, extensions)
3. ✗ Disabling the hook by renaming/deleting the before-create.sql file (hook is hardcoded)
4. ✗ Using `LOAD 'pgcrypto'` to load library directly (doesn't create SQL functions)
5. ✗ Changing file permissions on before-create.sql (not a file permission issue)
6. ✗ Moving pgcrypto creation to different init script timing
