# Story 2.1: Create Ping via Chat Interface

## Epic Context

This story is part of **Epic 2: Chat-First Ticket Creation & Threading**, which delivers EasyPing's core differentiator - a conversational, chat-first ticket creation experience that feels like messaging rather than form-filling. The epic enables users to describe issues naturally, engage in threaded conversations with agents, attach files, and track ticket status. This story specifically implements the initial ticket creation flow where users type their issue and the system creates a ticket in the database.

**Epic Goal:** Make EasyPing usable for basic support workflows with natural conversation flows (AI enhancements come in Epic 3).

## Story

**As an** end user,
**I want** to create a ping by typing my issue in a chat-style interface,
**so that** I can report problems naturally without filling forms.

## Acceptance Criteria

**Note:** This story implements the basic chat-first ping creation flow. AI features (clarification questions, problem statement summaries, category assignment) from the Epic 2.1 requirements in the PRD are deferred to Epic 3 (AI-Powered Features). This story focuses on the core ping creation mechanics.

1. "Create Ping" button navigates to chat-style ping creation screen
2. Interface shows message input box with placeholder: "Send a ping... describe your issue"
3. User types message and hits Enter or clicks Send button
4. New ping created in database with status=New, user message as first thread entry
5. Ping assigned auto-generated ID (e.g., #PING-001)
6. User immediately sees their message in conversation thread
7. Empty state for "My Pings" replaced with new ping in conversation list
8. Ping conversation list shows: last message preview, timestamp, unread indicator

### Task 7: Create Database Migration for Ping Number Generation (AC: 5)

**PURPOSE:** Ensure ping numbers auto-increment per tenant starting from 1.

- [x] Create migration file: `packages/database/migrations/{timestamp}_add_ping_number_sequence.sql`
  ```sql
  -- Create function to generate sequential ping numbers per tenant
  CREATE OR REPLACE FUNCTION generate_ping_number()
  RETURNS TRIGGER AS $$
  DECLARE
    next_number INTEGER;
  BEGIN
    -- Get the max ping_number for this tenant and add 1
    SELECT COALESCE(MAX(ping_number), 0) + 1
    INTO next_number
    FROM pings
    WHERE tenant_id = NEW.tenant_id;

    NEW.ping_number = next_number;
    RETURN NEW;
  END;
  $$ LANGUAGE plpgsql;

  -- Create trigger to auto-generate ping_number on insert
  CREATE TRIGGER set_ping_number
    BEFORE INSERT ON pings
    FOR EACH ROW
    EXECUTE FUNCTION generate_ping_number();
  ```

**API Patterns** [Source: docs/architecture/api-specification.md]

Next.js API route pattern:
```typescript
// apps/web/src/app/api/pings/route.ts
import { createClient } from '@/lib/supabase/server';
import { NextRequest, NextResponse } from 'next/server';
import { z } from 'zod';

export async function POST(request: NextRequest) {
  const supabase = await createClient();

  // Authenticate
  const { data: { user }, error: authError } = await supabase.auth.getUser();
  if (authError || !user) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
  }

  // Validate request
  const body = await request.json();
  const validation = schema.safeParse(body);
  if (!validation.success) {
    return NextResponse.json({ error: validation.error }, { status: 400 });
  }

  // Create ping
  const { data: ping, error } = await supabase
    .from('pings')
    .insert({ ... })
    .select()
    .single();

  if (error) {
    return NextResponse.json({ error: 'Failed to create ping' }, { status: 500 });
  }

  return NextResponse.json({ ping }, { status: 201 });
}
```
