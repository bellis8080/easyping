# Story 2.3: File Attachments in Conversations

## Epic Context

This story is part of **Epic 2: Chat-First Ticket Creation & Threading**, which delivers EasyPing's core differentiator - a conversational, chat-first ticket creation experience that feels like messaging rather than form-filling. The epic enables users to describe issues naturally, engage in threaded conversations with agents, attach files, and track ticket status.

**Epic Goal:** Make EasyPing usable for basic support workflows with natural conversation flows (AI enhancements come in Epic 3).

## Status

Done

## Story

**As a** user or agent,
**I want** to attach files (screenshots, logs, documents) to ping conversations,
**so that** I can provide context and evidence for issues.

## Acceptance Criteria

1. Message input box includes attachment button (paperclip icon)
2. Clicking attachment opens file picker (supports images, PDFs, text files, logs)
3. Selected files uploaded to Supabase Storage
4. File attachment appears in conversation thread with filename and file size
5. Image attachments display inline thumbnails (clickable to view full size)
6. Non-image files show download link with icon indicating file type
7. File size limited to 10MB per attachment
8. Multiple files can be attached to single message (up to 5 files)
9. Upload progress indicator shown during file upload

    ```typescript
    async function uploadFiles(
      files: File[],
      userId: string
    ): Promise<Array<{ name: string; path: string; size: number; type: string }>> {
      const uploadedFiles = [];

      for (const file of files) {
        const filePath = `${userId}/${Date.now()}-${file.name}`;

        // Upload to Supabase Storage
        const { data, error } = await supabase.storage
          .from('ping-attachments')
          .upload(filePath, file, {
            cacheControl: '3600',
            upsert: false,
          });

        if (error) {
          throw new Error(`Failed to upload ${file.name}: ${error.message}`);
        }

        // Track upload progress (simplified)
        setUploadProgress((prev) => ({ ...prev, [file.name]: 100 }));

        uploadedFiles.push({
          name: file.name,
          path: data.path,
          size: file.size,
          type: file.type,
        });
      }

      return uploadedFiles;
    }
    ```

  import { FileText, File, FileArchive, FileImage, FileVideo } from 'lucide-react';

  export function formatFileSize(bytes: number): string {
    if (bytes === 0) return '0 Bytes';
    const k = 1024;
    const sizes = ['Bytes', 'KB', 'MB', 'GB'];
    const i = Math.floor(Math.log(bytes) / Math.log(k));
    return `${Math.round(bytes / Math.pow(k, i) * 100) / 100} ${sizes[i]}`;
  }

  export function getFileIcon(mimeType: string) {
    if (mimeType.startsWith('image/')) return FileImage;
    if (mimeType.startsWith('video/')) return FileVideo;
    if (mimeType === 'application/pdf') return FileText;
    if (mimeType.includes('zip') || mimeType.includes('archive')) return FileArchive;
    return File;
  }

  export function isImageFile(mimeType: string): boolean {
    return mimeType.startsWith('image/');
  }

  export const ACCEPTED_FILE_TYPES = [
    'image/png',
    'image/jpeg',
    'image/gif',
    'image/webp',
    'application/pdf',
    'text/plain',
    'text/csv',
    'application/json',
    'application/xml',
    'application/zip',
  ];

  export const MAX_FILE_SIZE = 10485760; // 10MB in bytes

### File Upload Flow

**Client-Side Upload Process:**
1. User selects files via file picker
2. Validate file count (max 5), size (max 10MB), and type
3. Show file preview list with selected files
4. On send, upload files to Supabase Storage:
   - Path: `{user_id}/{timestamp}-{filename}`
   - Bucket: `ping-attachments`
   - Track upload progress per file
5. Create message via API with attachment metadata
6. Clear selected files and upload progress after success

**Server-Side Processing:**
1. Validate request body (message content + attachments array)
2. Create ping message in database
3. Insert attachment records in `ping_attachments` table
4. Return message with attachments to client
5. Realtime subscription broadcasts new message to other users

**File Access Control:**
- Supabase Storage RLS policies enforce user can only upload to their own folder
- Signed URLs provide temporary (1 hour) read access to files
- RLS policies on `ping_attachments` table ensure users only see attachments for pings they have access to
