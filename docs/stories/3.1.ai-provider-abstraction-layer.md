# Story 3.1: AI Provider Abstraction Layer

## Epic Context

This story is part of **Epic 3: AI Integration & Intelligent Routing**, which transforms EasyPing from basic ticketing to an AI-native support desk. The epic integrates multiple AI providers (OpenAI, Anthropic, Azure) with a bring-your-own-key model, automatically categorizes and routes incoming pings, generates AI summaries pinned at top of threads, and provides Echo (AI assistant) with suggested responses for agents.

**Epic Goal:** Deliver the core AI value proposition that differentiates EasyPing from traditional help desks.

## Acceptance Criteria

1. AI provider interface defined in `packages/ai` with methods: `categorize()`, `summarize()`, `suggestResponse()`, `generateEmbedding()`
2. OpenAI provider implementation (GPT-4o mini default, GPT-4 optional)
3. Anthropic provider implementation (Claude 3 Haiku default, Claude 3.5 Sonnet optional)
4. Azure OpenAI provider implementation
5. Provider configuration stored in database (encrypted API keys, model selection)
6. Graceful fallback when AI unavailable (error logged, feature disabled temporarily)
7. Provider selection configurable per organization in settings
8. Unit tests with mocked API responses (90%+ coverage)

### Key Technical Decisions

**Why Multiple Providers?**
- User choice: Different organizations prefer different AI vendors
- Cost optimization: Claude Haiku cheaper than GPT-4, users can choose
- Enterprise requirements: Some orgs require Azure-hosted models for compliance
- Vendor lock-in avoidance: Easy to swap providers if one raises prices or has outages

**Why BYOK (Bring Your Own Key)?**
- No centralized billing/subscription (simpler MVP)
- Users pay AI costs directly to provider (transparent pricing)
- No markup/middleman fees (community-friendly)
- Avoids rate limiting issues from shared keys

**Why These Default Models?**
- **GPT-4o mini:** Released July 2024, 60% cheaper than GPT-3.5-turbo with better performance, 128K context window vs 16K
- **Claude 3 Haiku:** Most cost-effective Anthropic model at $0.25/$1.25 per M tokens (4x cheaper than Claude 3.5 Haiku)
- **Research-backed:** Based on December 2024 analysis of support desk AI use cases and current pricing
- **User choice:** Organizations can upgrade to more powerful models (GPT-4, Claude 3.5 Sonnet) if they need higher quality and have budget

**Why Provider Abstraction?**
- Consistent API across providers (devs don't need to learn 3 different SDKs)
- Easy to add new providers later (implement interface, update factory)
- Testing simplified (mock one interface instead of three SDKs)
- Future-proof for new AI vendors (e.g., Cohere, Mistral, local models)
