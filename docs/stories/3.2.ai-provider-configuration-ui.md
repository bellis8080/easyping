# Story 3.2: AI Provider Configuration UI

## Epic Context

This story is part of **Epic 3: AI Integration & Intelligent Routing**, which transforms EasyPing from basic ticketing to an AI-native support desk. The epic integrates multiple AI providers (OpenAI, Anthropic, Azure) with a bring-your-own-key model, automatically categorizes and routes incoming pings, generates AI summaries pinned at top of threads, and provides Echo (AI assistant) with suggested responses for agents.

**Epic Goal:** Deliver the core AI value proposition that differentiates EasyPing from traditional help desks.

## Story

**As an** organization owner,
**I want** to configure my AI provider API keys and model preferences,
**so that** AI features work with my own API account.

## Acceptance Criteria

1. Settings page includes "AI Configuration" section
2. Form to input API key for OpenAI, Anthropic, or Azure OpenAI
3. Provider selection dropdown (OpenAI, Anthropic, Azure)
4. Model selection dropdown (GPT-4o mini, GPT-4, Claude 3 Haiku, Claude 3.5 Sonnet, etc.)
5. "Test Connection" button validates API key and shows success/error message
6. API keys encrypted before storing in database
7. Option to disable AI features if no key configured
8. Warning shown when AI credits low (if provider API returns usage info)

### Architecture Context

**Tech Stack** [Source: docs/architecture/tech-stack.md]
- **Frontend Framework:** Next.js 14+ App Router (NOT Pages Router)
- **UI Components:** shadcn/ui + Radix UI + Tailwind CSS
- **Forms:** React Hook Form + Zod validation
- **State:** React Hooks (useState, useEffect) for component state
- **Database:** Supabase (PostgreSQL 15+) with pgcrypto extension for encryption
- **Testing:** Vitest for unit/integration tests
- **Icons:** Lucide React

**Database Schema** [Source: docs/architecture/database-schema.md + supabase/migrations/20251204000001_add_ai_configuration.sql]
- **Organizations table** has `ai_config` JSONB column with schema:
  ```typescript
  {
    provider?: 'openai' | 'anthropic' | 'azure';
    encrypted_api_key?: string;
    model?: string;
    enabled?: boolean;
  }
  ```
- **Encryption functions:**
  - `encrypt_api_key(api_key TEXT, org_id UUID) RETURNS TEXT` - Encrypts API key with org ID as salt
  - `decrypt_api_key(encrypted_key TEXT, org_id UUID) RETURNS TEXT` - Decrypts API key, returns NULL on failure
- **RLS:** Intentionally DISABLED. Tenant isolation enforced at application layer via `tenant_id` filters.

**AI Provider Details** [Source: docs/stories/3.1.ai-provider-abstraction-layer.md]
- **OpenAI Models:**
  - GPT-4o mini (default): $0.15/$0.60 per M tokens (60% cheaper than GPT-3.5-turbo)
  - GPT-4: Higher quality, more expensive
  - GPT-4 Turbo: Fast GPT-4 variant
- **Anthropic Models:**
  - Claude 3 Haiku (default): $0.25/$1.25 per M tokens (most cost-effective)
  - Claude 3.5 Sonnet: $2/$10 per M tokens (higher quality)
  - Claude 3 Opus: Highest quality, most expensive
  - **LIMITATION:** No embeddings API (organizations must use OpenAI for semantic search)
- **Azure OpenAI:**
  - Custom deployment names (user-provided)
  - Same models as OpenAI but Azure-hosted
- **Provider Factory:** `createAIProvider(providerType, config)` from `packages/ai/src/factory.ts`

### Important Reminders

1. **Apply Migration First:** Story 3.1 created the migration but didn't apply it. You MUST run `npx supabase db push --local` before implementing this story.
2. **Two UI Entry Points:** AI configuration can be set in TWO places:
   - **Setup Wizard** (first-run): Step 3 of setup wizard (`AIConfigStep` component) - optional, can skip
   - **Settings Page** (post-setup): `/dashboard/settings/ai` - owner-only access for changes
3. **Setup Wizard Integration:** Verify the setup wizard API route (`/api/setup`) properly handles AI config from wizard and encrypts API keys.
4. **RLS Disabled:** RLS is intentionally disabled. DO NOT re-enable RLS policies. Use application-layer tenant isolation via `tenant_id` filters.
5. **Encryption Required:** API keys MUST be encrypted using `encrypt_api_key()` before storage. Never store plain-text API keys.
6. **Owner-Only Access:** Only users with `role = 'owner'` can access and modify AI configuration in settings. Enforce in both UI and API routes.
7. **Anthropic Limitation:** When Anthropic is selected, show warning about embeddings limitation in BOTH setup wizard and settings. Organizations will need OpenAI for semantic search.
8. **Cost Optimization:** Default to cost-optimized models (GPT-4o mini, Claude 3 Haiku) and show cost comparison tooltips in BOTH UIs.
9. **Error Handling:** Use graceful fallback from Story 3.1. Display AI feature status if auto-disabled after failures.
10. **TypeScript Strict Mode:** No `any` types without justification. Explicit return types for all exported functions.
11. **Ubiquitous Language:** Use "organization" not "tenant" in user-facing text. Use "ping" not "ticket" in all code and UI.
12. **Test Coverage:** 90%+ coverage required for new code. Co-locate tests with source files.
