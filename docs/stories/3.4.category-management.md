## Story

**As an** owner setting up EasyPing,
**I want** Echo to interview me about my organization's support needs during setup,
**so that** the system understands what types of issues we handle and can suggest relevant categories.

**As a** manager or owner,
**I want** to add, edit, archive, and reorder ticket categories,
**so that** I can customize categories to match my organization's support structure.

## Acceptance Criteria

### Part A: Organization Support Profile (Setup Wizard Extension)

1. Setup wizard includes new "Support Profile" step after AI Configuration
2. **If AI is configured:** Echo interviews the owner conversationally to understand:
   - What type of organization (IT department, HR, customer service, facilities, etc.)
   - What kinds of issues/requests the support team handles
   - Who the typical users are (employees, customers, partners)
   - Any specific systems, tools, or processes they support
3. **If AI is not configured:** Simple form with:
   - Support type dropdown (IT Support, HR Support, Customer Service, Facilities, General, Other)
   - Textarea for "Describe what your support team handles"
4. Echo generates a structured support profile from the conversation
5. Owner can review and edit the generated profile before proceeding
6. Support profile stored in `organizations.support_profile` JSONB column
7. Setup wizard includes new "Categories" step after Support Profile
8. AI suggests relevant categories based on the support profile (if AI configured)
9. Owner can accept, reject, or modify suggested categories
10. Owner can add custom categories during setup
11. Skipping profile step falls back to generic 7 default categories

### Part B: Category Management (Settings Page)

12. Settings page includes "Categories" section (manager/owner only)
13. Categories list shows: Category name, color (for badge), ticket count, active/archived status
14. "Add Category" button opens form: Name (required), Description (optional), Color picker, Icon (optional)
15. Manager can edit existing category name, description, color, and icon
16. Manager can archive category (soft-delete) - tickets keep category, but not available for new tickets
17. Cannot delete category if tickets currently use it (must archive instead)
18. "Other" category is system-reserved and cannot be deleted (can be renamed)
19. Category changes immediately reflected in ticket creation, routing rules, and analytics
20. Maximum 50 active categories per organization (prevent overwhelming dropdown)
21. Categories sortable via drag-and-drop to control display order

### Part C: Support Profile Management (Settings Page)

22. Settings page includes "Support Profile" section showing current profile
23. Owner can edit the support profile manually (textarea + structured fields)
24. "Re-interview with Echo" button launches conversational profile update (if AI configured)
25. Profile changes can trigger category suggestions (optional - owner chooses to apply or ignore)

### 2025-12-12: Echo Support Profile Integration

**Issue Found During QA:**
Echo AI in pings was assuming everything was an IT support issue regardless of the organization's actual support profile. The AI prompts were hardcoded with IT-specific examples (VPN, printers, laptops) without referencing the organization's support context.

**Root Cause:**
- `echo-conversation-service.ts` had hardcoded prompts assuming IT support
- Echo API routes (`/echo/start`, `/echo/continue`) fetched `ai_config` but not `support_profile`
- System prompts said "You are Echo, an AI assistant for IT support" regardless of actual org type

**Fix Applied:**

1. **Updated `echo-conversation-service.ts`:**
   - Added `SupportProfileContext` interface to accept support profile data
   - Added `buildSupportContext()` helper to build context from profile
   - Updated `analyzeConversation()` to use org's support type, description, and common issues
   - Updated `generateProblemStatement()` to use support context
   - Made prompts recognize that not everything is an "issue" - users may have questions, requests, or feedback
   - Dynamic system prompts: `"You are Echo, an AI assistant for ${ctx.supportType}"`

2. **Updated Echo API routes:**
   - `apps/web/src/app/api/pings/[pingNumber]/echo/start/route.ts` - Now fetches `support_profile` along with `ai_config`
   - `apps/web/src/app/api/pings/[pingNumber]/echo/continue/route.ts` - Same change

3. **Flexible statement formats:**
   - Issues: "User struggles with [problem]..."
   - Questions: "User wants to know [topic]..."
   - Requests: "User requests [action]..."
   - Feedback: "User provides feedback about [topic]..."

**Files Modified:**
- `apps/web/src/lib/services/echo-conversation-service.ts`
- `apps/web/src/app/api/pings/[pingNumber]/echo/start/route.ts`
- `apps/web/src/app/api/pings/[pingNumber]/echo/continue/route.ts`

**Verification:**
- ✅ TypeScript typecheck passes
- ✅ Echo now uses organization's support profile context in conversations
- ✅ QA testing confirmed correct behavior with Product Support profile
