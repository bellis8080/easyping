## Story

**As an** agent or end user,
**I want** the AI summary to automatically update as the conversation progresses,
**so that** I always have current context without manually refreshing or reading the entire thread.

## Current State

The pinned summary UI **already exists** and works:
- Summary displays at top of ping detail view ([ping-detail.tsx:618-643](apps/web/src/components/pings/ping-detail.tsx#L618-L643))
- Collapsible behavior implemented
- `pings.ai_summary` column stores the summary

**What's missing:** The summary is only set once during Echo finalization (as the problem statement). It never updates as the conversation continues between user and agent.

## Acceptance Criteria

### Part A: Automatic Summary Updates

1. Summary regenerated when 3+ new messages are added after the last summary update
2. Summary regenerated when ping status changes (new → in_progress, in_progress → resolved, etc.)
3. Summary content evolves to reflect:
   - Original issue description
   - Key updates from the conversation
   - Current resolution status
   - Any solutions attempted or provided
4. New `summary_updated_at` timestamp tracks when summary was last generated

### Part B: Manual Refresh

5. "Refresh" button added to summary header
6. Clicking refresh regenerates summary immediately
7. Loading state shown during regeneration (spinner)
8. Toast notification on successful refresh

### Part C: Summary Generation Logic

9. Summary prompt includes full conversation context (not just problem statement)
10. Summary limited to 500 characters maximum
11. Summary format includes:
    - Issue: What the user needs (1-2 sentences)
    - Progress: What's been discussed/tried (1-2 sentences)
    - Status: Current state and next steps
12. If AI unavailable, existing summary preserved (don't clear it)

### Summary Prompt Template

```
You are summarizing an ongoing support conversation for an agent.

Conversation:
{messages formatted as User/Agent/Echo/System}

Ping Context:
- Status: {status}
- Category: {category}
- Priority: {priority}

Generate a concise summary (max 500 chars) with these sections:

**Issue:** What the user needs help with (1-2 sentences)
**Progress:** What's been discussed or tried (1-2 sentences)
**Status:** Current state and suggested next steps

Keep it factual and actionable. Focus on information an agent needs to quickly understand the situation.
```

### Fire-and-Forget Pattern

For automatic regeneration (on new messages / status change), use fire-and-forget:
```typescript
// Don't await - let it run in background
generateAndUpdateSummary(pingId, tenantId).catch(console.error);
```

This ensures the primary operation (sending message, changing status) isn't blocked.
