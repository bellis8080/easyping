**Why This Story:**
After agents can document resolution steps (4.2.1), we can now generate high-quality KB articles from resolved pings. The AI uses both the public conversation AND private notes to create comprehensive documentation.

## Story

**As an** agent,
**I want** to generate a KB article draft when resolving a ping,
**so that** solutions can be captured and reused for future similar issues.

## Acceptance Criteria

1. Resolve confirmation modal shows checkbox: "Generate KB article from this ping"
2. When opted in, AI generates KB article draft with TWO resolution sections:
   - **User Resolution** (public): Self-service steps end users can perform, OR "Contact support" if agent intervention required
   - **Agent Resolution** (internal): Technical steps from private notes, visible only to agents
3. Generated articles include: title, public content, internal content, and suggested category
4. Draft is saved to `kb_articles` table with `status='draft'` and `source_ping_id` linked
5. Agent is redirected to KB editor to review/edit the draft (Story 4.3 will implement editor)
6. If AI generation fails, show error toast but still allow ping resolution
7. Generation happens asynchronously (non-blocking UI)
8. End users see only the public content; agents see both public + internal sections
9. Internal content stored in new `agent_content` column (not visible via public API)

### Dual-Resolution Concept

KB articles now have TWO resolution sections:

1. **User Resolution (`content`)** - What end users see:
   - Self-service steps they can perform themselves
   - OR "Contact support for assistance" if agent intervention required
   - Written in user-friendly, non-technical language

2. **Agent Resolution (`agent_content`)** - What agents see:
   - Technical steps derived from private notes
   - Backend fixes, configuration changes, etc.
   - Helps agents resolve similar issues faster

**Example:**

- **User Resolution:** "To reset your password, click 'Forgot Password' on the login page..."
- **Agent Resolution:** "User's account was locked due to failed login attempts. Unlocked via admin panel > Users > Reset Lock. Also cleared cached session in Redis."

### AI Prompt Strategy

The prompt should generate BOTH sections:

```typescript
const prompt = `You are a technical writer creating a knowledge base article from a resolved support conversation.

## Source Conversation (User's Issue):
${conversationText}

## Resolution Notes (Agent's Internal Notes):
${privateNotesText}

Generate a KB article with TWO resolution sections:

1. **Title**: A clear, searchable title describing the problem (max 100 chars)

2. **User Resolution (content)**: Public-facing content for end users
   - Problem description (from user's perspective)
   - Self-service solution steps IF the user can resolve this themselves
   - If agent intervention was required, write: "This issue requires support assistance. Please contact our support team."
   - Written in clear, non-technical language

3. **Agent Resolution (agentContent)**: Internal content for support agents
   - Technical steps taken to resolve the issue
   - Backend changes, configurations, or fixes applied
   - Tips for handling similar issues
   - This section is NEVER shown to end users

4. **Category**: Suggest a category slug from: ${availableCategories}

Respond in JSON format:
{
  "title": "...",
  "content": "...",
  "agentContent": "...",
  "suggestedCategorySlug": "..."
}`;
```

### Filtering Agent Content

```typescript
// apps/web/src/lib/utils/kb-article-filter.ts
export function filterKBArticleForUser(
  article: KBArticle,
  userRole: UserRole
): KBArticle {
  // End users don't see agent_content
  if (userRole === 'end_user') {
    const { agent_content, ...publicArticle } = article;
    return publicArticle as KBArticle;
  }
  // Agents, managers, owners see everything
  return article;
}
```

### Slug Generation Utility

```typescript
// apps/web/src/lib/utils/slug.ts
export function generateSlug(title: string): string {
  return title
    .toLowerCase()
    .trim()
    .replace(/[^a-z0-9\s-]/g, '') // Remove special chars
    .replace(/\s+/g, '-')          // Replace spaces with hyphens
    .replace(/-+/g, '-')           // Remove consecutive hyphens
    .substring(0, 100);            // Limit length
}
```
