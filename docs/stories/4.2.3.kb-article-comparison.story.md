**Why This Story:**
Not every resolved ping represents a new problem. Many are variations of existing KB content. This story adds intelligence to compare resolved pings against the existing KB, and when a match is found, creates a "smart merge" draft that enhances the original article with new information.

## Story

**As an** agent,
**I want** the system to compare my resolved ping against existing KB articles,
**so that** I can enhance existing articles instead of creating duplicates.

## Acceptance Criteria

1. Before generating a new KB article, system searches existing KB for similar content
2. If similar article found (>70% semantic match), show agent the match with options:
   - "Enhance existing article" - AI merges new info into existing
   - "Create new article anyway" - proceeds with new article creation
   - "Skip KB generation" - just resolve, no KB action
3. "Enhance existing" creates a NEW draft version (preserves original until approved)
4. Enhanced draft includes diff-style view showing what changed
5. When agent approves enhanced draft, it REPLACES the original article
6. Original article's view_count, helpful_count preserved after replacement
7. If no similar article found, proceed with normal new article generation (Story 4.2.2)
8. Similarity search uses category as primary filter, then content matching

## Dev Notes

### Similarity Search Strategy (Pre-pgvector)

Until Story 4.4 implements pgvector semantic search, use PostgreSQL full-text search:

```sql
-- Find similar articles using full-text search
SELECT
  id,
  title,
  content,
  ts_rank(to_tsvector('english', title || ' ' || content), plainto_tsquery('english', $1)) as similarity
FROM kb_articles
WHERE tenant_id = $2
  AND status = 'published'
  AND deleted_at IS NULL
  AND to_tsvector('english', title || ' ' || content) @@ plainto_tsquery('english', $1)
ORDER BY similarity DESC
LIMIT 5;
```

The similarity score (0-1) can be normalized to a percentage (0-100).

### Approval/Replacement Flow

```typescript
// When agent approves enhancement
async function approveEnhancement(draftId: string, supabaseAdmin: Client) {
  const draft = await getDraft(draftId);
  const original = await getArticle(draft.enhances_article_id);

  // 1. Copy metrics to draft
  await supabaseAdmin.from('kb_articles')
    .update({
      view_count: original.view_count,
      helpful_count: original.helpful_count,
      not_helpful_count: original.not_helpful_count,
      status: 'published',
      published_at: new Date().toISOString(),
      published_by: userId,
      enhances_article_id: null, // Clear the link
      slug: original.slug, // Take over the original slug
    })
    .eq('id', draftId);

  // 2. Archive original
  await supabaseAdmin.from('kb_articles')
    .update({
      status: 'archived',
      slug: original.slug + '-archived-' + Date.now(),
    })
    .eq('id', original.id);
}
```

### Database Migration

```sql
-- Add enhancement tracking to kb_articles
ALTER TABLE kb_articles
ADD COLUMN enhances_article_id UUID REFERENCES kb_articles(id) ON DELETE SET NULL;

-- Index for finding enhancement drafts
```
