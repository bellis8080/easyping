**Why This Story:**
Stories 4.2.1-4.2.3 generate draft KB articles from resolved pings, but agents cannot yet manage these drafts. This story builds the KB management interface allowing agents to review, edit, and publish KB articles. Without this, generated drafts remain unpublished and invisible to end users.

**Critical Testing Dependency:**
Story 4.2.3's comparison flow UI (showing "Enhance Existing", "Create New", "Skip KB" buttons) requires published KB articles to function. This story enables publishing, so after completion, return to test Story 4.2.3's comparison flow.

## Story

**As an** agent,
**I want** to review, edit, and publish KB article drafts,
**so that** I can ensure quality before making articles public.

## Acceptance Criteria

1. KB section in sidebar navigation (agents only)
2. KB dashboard shows tabs: Published, Drafts, Archived
3. Draft articles list shows title, category, source ticket, created date
4. Clicking draft opens editor with markdown support (WYSIWYG)
5. Editor toolbar includes: bold, italic, headers, lists, code blocks, links, images
6. WYSIWYG editor shows formatted content as user types (no separate preview pane)
7. "Publish" button changes status to published and makes visible to users
8. "Save Draft" button saves without publishing
9. "Delete" button soft-deletes article
10. Published articles display publish date and author name

### IMPORTANT: RLS is Disabled on Purpose

**DO NOT re-enable Row Level Security (RLS) on these tables.**

RLS is intentionally disabled on several tables (including `kb_articles`) because Supabase Realtime does not work properly with RLS enabled. Security is enforced at the API layer:

1. **Authentication:** Verify user via `supabase.auth.getUser()`
2. **Authorization:** Check user role via `canViewPrivateMessages()` helper
3. **Tenant Isolation:** Filter by `tenant_id` from user's profile in all queries
4. **Admin Client:** Use `createAdminClient()` only AFTER validation passes

This is documented in CLAUDE.md under "Common Gotchas" and in migration file comments.

### Slug Generation

```typescript
function generateSlug(title: string): string {
  return title
    .toLowerCase()
    .replace(/[^a-z0-9\s-]/g, '') // Remove special chars
    .replace(/\s+/g, '-')          // Spaces to hyphens
    .replace(/-+/g, '-')           // Multiple hyphens to single
    .replace(/^-|-$/g, '');        // Trim leading/trailing hyphens
}
```

### WYSIWYG Editor Implementation

Use Tiptap - a headless, framework-agnostic rich text editor built on ProseMirror.

**Storage Format:** Markdown (not HTML)

```bash
pnpm --filter @easyping/web add @tiptap/react @tiptap/starter-kit @tiptap/extension-link @tiptap/extension-image @tiptap/extension-code-block-lowlight turndown
```

```tsx
import { useEditor, EditorContent } from '@tiptap/react';
import StarterKit from '@tiptap/starter-kit';
import Link from '@tiptap/extension-link';
import Image from '@tiptap/extension-image';
import TurndownService from 'turndown';

// Convert HTML to Markdown on save
const turndown = new TurndownService({
  headingStyle: 'atx',
  codeBlockStyle: 'fenced',
});

function KBEditor({ content, onChange }: { content: string; onChange: (markdown: string) => void }) {
  const editor = useEditor({
    extensions: [
      StarterKit,
      Link.configure({ openOnClick: false }),
      Image,
    ],
    content, // Can accept markdown or HTML for initial content
    onUpdate: ({ editor }) => {
      // Convert to markdown before saving
      const html = editor.getHTML();
      const markdown = turndown.turndown(html);
      onChange(markdown);
    },
  });

  if (!editor) return null;

  return (
    <div className="border rounded-lg">
      {/* Toolbar */}
      <div className="flex gap-1 border-b p-2 bg-slate-50">
        <Button
          variant="ghost"
          size="sm"
          onClick={() => editor.chain().focus().toggleBold().run()}
          className={editor.isActive('bold') ? 'bg-slate-200' : ''}
        >
          <Bold className="h-4 w-4" />
        </Button>
        <Button
          variant="ghost"
          size="sm"
          onClick={() => editor.chain().focus().toggleItalic().run()}
          className={editor.isActive('italic') ? 'bg-slate-200' : ''}
        >
          <Italic className="h-4 w-4" />
        </Button>
        {/* More toolbar buttons for headings, lists, code, link, image */}
      </div>

      {/* Editor content - shows formatted output directly */}
      <EditorContent editor={editor} className="prose prose-slate max-w-none p-4 min-h-[400px]" />
    </div>
  );
}
```

**Why Tiptap + Markdown:**
- Headless architecture works well with shadcn/ui and Tailwind
- Built on ProseMirror (battle-tested)
- Markdown storage is portable and future-proof
- `turndown` library converts HTML to clean markdown on save
- Existing KB articles from Story 4.2.2 already store markdown
- Extensible with custom extensions
- Good TypeScript support
