**Why This Story:**
The KB browse page is the primary entry point for users to discover and browse published articles. It must exist before:
- Story 4.4 can add semantic search to it
- Story 4.5 can link from article cards to detail pages
- Users can self-serve by browsing the knowledge base

## Story

**As a** user or agent,
**I want** to browse the knowledge base with category filtering and see popular articles,
**so that** I can discover relevant support articles and self-serve before creating a ping.

## Acceptance Criteria

1. `/kb` route displays the Knowledge Base browse page accessible from sidebar navigation
2. Published articles displayed as cards showing: title, snippet (first 150 chars of content), category badge, view count, helpfulness percentage, last updated date
3. Categories sidebar (right side) shows all categories with article counts, clicking filters the list
4. "All" category option shows all published articles (default view)
5. Popular Articles widget shows top 5 articles by view count
6. Search bar at top with placeholder "Search articles, guides, and FAQs..."
7. Search uses full-text search (from Story 4.1) with debounced input (300ms)
8. Results show count: "X articles found"
9. Empty state when no articles: "No articles published yet. Check back soon!"
10. Empty search results: "No articles found. Try different keywords or create a ping."
11. Articles sorted by most recently updated by default
12. Clicking an article card navigates to `/kb/[slug]` (detail page, Story 4.5)
13. KB page accessible to all authenticated users (end users, agents, managers, owners)
14. End users see only public content; internal/agent-only articles hidden

### Database Schema (from Story 4.1)

```sql
-- kb_articles table columns relevant to this story:
id UUID
tenant_id UUID
title TEXT
slug TEXT (unique per tenant)
content TEXT (markdown)
category_id UUID (FK to categories)
status TEXT ('draft', 'published', 'archived')
view_count INTEGER DEFAULT 0
helpful_count INTEGER DEFAULT 0
not_helpful_count INTEGER DEFAULT 0
updated_at TIMESTAMPTZ
deleted_at TIMESTAMPTZ (soft delete)
search_vector TSVECTOR (full-text search)
```

### Full-Text Search Pattern

```typescript
// Use PostgreSQL plainto_tsquery for search
const { data, error } = await supabase
  .from('kb_articles')
  .select(`
    id, title, slug, content, category_id,
    view_count, helpful_count, not_helpful_count,
    updated_at,
    categories (id, name, color)
  `)
  .eq('tenant_id', tenantId)
  .eq('status', 'published')
  .is('deleted_at', null)
  .textSearch('search_vector', query, { type: 'plain' })
  .order('updated_at', { ascending: false })
  .limit(20);
```

### Helpfulness Percentage Calculation

```typescript
function calculateHelpfulness(helpful: number, notHelpful: number): number {
  const total = helpful + notHelpful;
  if (total === 0) return 0;
  return Math.round((helpful / total) * 100);
}

// Display: "95% helpful" or "No ratings yet" if total is 0
```

### URL Query Parameters

| Param | Purpose | Example |
|-------|---------|---------|
| `q` | Search query | `/kb?q=password+reset` |
| `category` | Filter by category slug | `/kb?category=account` |
| Both | Combined filter | `/kb?q=login&category=security` |
