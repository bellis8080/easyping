**What We're Building:**
This story adds semantic search capabilities to the Knowledge Base using pgvector embeddings. Users can search using natural language (e.g., "my screen keeps flickering") and find relevant articles even without exact keyword matches. This builds on:
- The `embedding` VECTOR(1536) column already created in Story 4.1
- The IVFFlat index already created for cosine similarity search
- The `generateEmbedding()` method already implemented in `@easyping/ai` package
- The KB browse page search UI from Story 4.3.5

**Key Integration Points:**
1. Generate embeddings when articles are published (hook into publish workflow)
2. Enhance the existing `/api/kb/public/articles` endpoint with semantic search
3. Use full-text search as fallback when embeddings unavailable

## Story

**As a** user or agent,
**I want** to search KB articles using natural language,
**so that** I can find solutions even if I don't know exact keywords.

## Acceptance Criteria

1. `kb_articles` table includes `embedding` column (vector type from pgvector) - **Already exists from Story 4.1**
2. When article published, AI generates embedding from title + content
3. Search query generates embedding and finds similar articles (cosine similarity)
4. Search returns top 10 most relevant articles ranked by similarity
5. Search also includes traditional full-text search as fallback
6. Results show: title, excerpt with highlighted keywords, category badge, view count
7. Search box in KB page with instant results (debounced, updates as user types)
8. Empty state if no results: "No articles found. Try different keywords or create a ping."

### AI Provider Integration

The `@easyping/ai` package already has embedding generation implemented:

```typescript
// packages/ai/src/providers/openai.ts
async generateEmbedding(text: string): Promise<number[]> {
  const response = await this.client.embeddings.create({
    model: 'text-embedding-ada-002',
    input: text,
  });
  return response.data[0]?.embedding;
}
```

**Important Notes:**
- Model: `text-embedding-ada-002` returns 1536-dimension vectors (matches DB column)
- Cost: ~$0.0001 per 1K tokens (very cheap)
- Token limit: ~8191 tokens per request (~32K characters)
- Truncate long articles to stay within limits

[Source: packages/ai/src/providers/openai.ts]

### pgvector Cosine Similarity Query

```sql
-- Find similar articles using cosine distance
SELECT id, title, content,
       1 - (embedding <=> $1) as similarity
FROM kb_articles
WHERE tenant_id = $2
  AND status = 'published'
  AND deleted_at IS NULL
  AND embedding IS NOT NULL
ORDER BY embedding <=> $1
LIMIT 10;
```

**Distance Operators:**
- `<=>` - Cosine distance (use for text embeddings)
- `<->` - L2 distance (Euclidean)
- `<#>` - Inner product

[Source: docs/architecture/database-schema.md]
