## Story

**As an** agent resolving a ping,
**I want** the Echo panel to suggest relevant KB articles based on the conversation,
**So that** I can quickly share helpful resources with end users without manually searching the knowledge base.

---

## Acceptance Criteria

1. **Echo panel shows "Related KB Articles" section** below response suggestions
2. **AI searches KB using semantic search** (embeddings) based on ping title + AI summary
3. **Top 3 most relevant published articles** displayed with title and short preview
4. **Clicking an article opens a modal overlay** showing full article content
5. **"Insert Link" button** adds a KB article reference to the agent's response
6. **"No relevant articles found" message** displayed when no matches above threshold
7. **KB suggestions refresh** when AI summary is updated
8. **Only published articles shown** (not drafts or archived)

```typescript
  .eq('ping_number', params.pingNumber)
  .single();

```

**KB Suggestions Pattern** (from `api/kb/suggestions/route.ts`):
```typescript
// Semantic search with embedding
const embedding = await generateEmbedding(query);
const { data: articles } = await adminClient.rpc('match_kb_articles', {
  query_embedding: embedding,
  similarity_threshold: 0.5,
  match_count: 3
});
```

```typescript
export function useEchoKBSuggestions(pingNumber: string) {
  const [suggestions, setSuggestions] = useState<KBSuggestion[]>([]);

    const res = await fetch(`/api/pings/${pingNumber}/kb-suggestions`, {
      credentials: 'include'

  return { suggestions, isLoading, refetch: fetchSuggestions };
}
```

### Important Notes

1. **Uses AI Summary for search** - The ping's `ai_summary` field (generated by Echo) provides a distilled version of the conversation, making semantic search more efficient and accurate than searching raw messages.

2. **RLS is intentionally disabled** on `kb_articles` table for Supabase Realtime compatibility. Auth is enforced at API layer.

3. **Credentials required** - Always use `credentials: 'include'` for authenticated API calls.

4. **Orange accent color** - KB links in Echo panel should use orange accent (`text-orange-600`) for consistency with other KB UI.

5. **Markdown rendering** - Use `marked` library (already installed) for rendering article content in modal.

6. **Echo panel location** - The Echo panel is inline in `ping-detail.tsx`, not a separate component. KB suggestions section should be added within the existing Echo panel structure.

7. **Embedding generation** - Reuse `generateEmbedding()` from `@easyping/ai` package.
