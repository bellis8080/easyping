**What We're Building:**
This story implements the core SLA time tracking functionality with visual UI display. Building on Story 5.1's SLA policies, we now track when key events occur (`first_response_at`, `resolved_at`) and implement the logic to pause/resume the resolution timer when a ping is "Waiting on User". We also add SLA timer UI components to the agent inbox so agents can see countdown timers with color-coded status indicators.

**Key Integration Points:**

1. Add tracking columns to `pings` table (`first_response_at`, `resolved_at`, `sla_paused_at`, `sla_paused_duration_minutes`)
2. Detect first agent response and record timestamp
3. Record resolution timestamp when ping status changes to "Resolved"
4. Pause resolution timer when status changes to "Waiting on User"
5. Resume timer when status changes back or agent replies
6. Calculate remaining time accounting for paused periods
7. **SLA Timer Badge** in ping list (shows most urgent timer with color)
8. **Dual SLA Timer Display** in ping detail toolbar (shows both timers)
9. Color coding: Green (>50%), Yellow (20-50%), Red (<20% or breached)

## Story

**As a** system,
**I want** to track time-to-first-response and time-to-resolution for pings,
**so that** SLA compliance can be measured accurately.

## Acceptance Criteria

1. `pings` table includes: `first_response_at`, `resolved_at` (Story 5.1 added `sla_first_response_due`, `sla_resolution_due`)
2. When ping created, **both SLA due times** are already calculated (implemented in Story 5.1)
3. **When first agent response posted:** `first_response_at` timestamp recorded (stops First Response SLA timer)
4. **When ping marked Resolved:** `resolved_at` timestamp recorded (stops Resolution SLA timer)
5. **Resolution SLA timer pauses** when ping status = "Waiting on User" (resumes when agent replies or status changes back to In Progress)
6. **First Response SLA timer never pauses** (always counts from ping creation to first agent message)
7. Business hours support: SLA calculations respect configured business hours (default: 24/7 for MVP)
8. **SLA timer calculations:**
   - Time remaining = sla_due_time - current_time (accounting for paused periods)
   - Breach status = current_time > sla_due_time
   - Color coding: Green (>50% time remaining), Yellow (20-50%), Red (<20% or breached)

### Pause/Resume Logic

**When status changes TO "Waiting on User":**
1. If not already paused (`sla_paused_at` is NULL)
2. Set `sla_paused_at = NOW()`
3. Resolution timer is now paused

**When status changes FROM "Waiting on User":**
1. Calculate pause duration: `NOW() - sla_paused_at`
2. Add duration to `sla_paused_duration_minutes`
3. Set `sla_paused_at = NULL` (resume timer)

**When calculating time remaining:**
```
effective_elapsed = NOW() - created_at - sla_paused_duration_minutes
if sla_paused_at is not NULL:
  effective_elapsed -= (NOW() - sla_paused_at)  // Subtract current pause
time_remaining = resolution_minutes - effective_elapsed
```
