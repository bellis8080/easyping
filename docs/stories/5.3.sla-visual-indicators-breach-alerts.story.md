**What We're Building:**
This story completes the SLA user experience by adding:
1. **Agent Features:** Sort inbox by SLA Risk, breach notification toasts
2. **End User Features:** Static response expectations (no timers/SLA terminology)

Story 5.2 already implemented the SLA timer badges and dual timer displays for agents. This story adds the finishing touches: sorting, breach alerts, and the end user experience.

**Key Integration Points:**

1. **Sort by SLA Risk** - Breached first, then at-risk (yellow), then on-track (green)
2. **Breach Toast Notifications** - Warning when entering red zone, error when breached
3. **End User Static Expectations** - "We typically respond within 2 hours" messages
4. **No SLA terminology for end users** - Internal tracking only

## Story

**As an** agent,
**I want** visual indicators showing both First Response and Resolution SLA status,
**so that** I can prioritize pings approaching breach.

## Acceptance Criteria

**Agent Views (Timers - Implemented in Story 5.2):**

1. [x] **Ticket list (inbox) shows most urgent SLA timer** (Story 5.2)
2. [x] **Ticket detail toolbar shows both timers** (Story 5.2)

**Agent Views (New for Story 5.3):**

5. **SLA breach notifications:**
   - In-app toast when timer enters red zone (<20% remaining)
   - In-app toast when SLA breached
   - System notification for assigned agent (browser notification)
   - Email notification when SLA breached (configurable in settings - future)

**End User Views (No Timers):**

6. **End users do NOT see SLA timers or countdown clocks** (verified in Story 5.2)

7. **Instead, show static expectations:**
   - At ping creation: "Ticket created! We typically respond within 2 hours."
   - In ping view: "Created 15 minutes ago" (already implemented)
   - After resolution: "Resolved in 2h 15m"

8. **No "SLA" terminology exposed to end users** (internal tracking only)
